# AI Shopping Assistant

AI-ассистент для подбора товаров по запросу пользователя. Система анализирует запросы пользователей и предоставляет персонализированные рекомендации товаров из каталога.

---

## AI Commercial Chatbot

AI-ассистент для подбора товаров с встраиваемым виджетом чата.

Проект помогает посетителю сайта подобрать товар через чат.

Он показывает виджет на сайте, принимает сообщение пользователя, ищет подходящие товары и показывает ответ в чате. Для просмотра работы есть демо-страница и панель с историей запросов.

---

## Задача, аудитория и боль

**Задача:** Ускорить подбор товаров и консультации в онлайн-магазине строительных товаров. Для задач клиентов система собирает корзины товаров "под ключ" - полный набор необходимых товаров для решения задачи.

**Для кого:** Покупатели строительных товаров, консультанты магазина, обрабатывающие поток запросов на подбор товаров.

**Боль сейчас:**

* Ручной поиск товаров в каталоге занимает значительное время
* Субъективность в рекомендациях товаров без понимания контекста запроса
* Долгий цикл от запроса до получения рекомендаций
* Высокая нагрузка на консультантов при росте объёма запросов
* Неоднозначные запросы пользователей требуют множественных уточняющих вопросов

## Что именно сделает PoC на демо

Бот реализует два клиентских флоу:

### Consultation Flow (Флоу консультации)

* Бот получает вопрос пользователя (например: «Что лучше для покраски стен, кисточка или валик?»)
* Генерирует экспертский ответ на вопрос через LLM на основе знаний о строительных товарах
* Параллельно выделяет сущности из запроса через LLM (например: «краска, кисточка, валик»)
* Выполняет поиск товаров в каталоге (лексический BM25 по текстовым полям каталога) по выделенным сущностям
* На выходе: экспертский ответ на вопрос + ссылки на найденные товары

### Problem Solving Flow (Флоу решения задачи)

* Бот получает описание задачи пользователя (например: «Хочу повесить шторы»)
* Анализирует задачу через LLM и задает уточняющие вопросы для критически важных параметров (например: «Из какого материала стена?»)
* После получения ответов определяет полный набор необходимых товаров через LLM
* Выполняет поиск в каталоге (лексический BM25) по каждому необходимому товару
* Формирует корзину товаров под ключ (от основных инструментов до вспомогательных материалов)
* На выходе: полный набор товаров для решения задачи, организованный по категориям

Каталог товаров для PoC хранится в SQLite и загружается из таблицы `products`.

### LLM-провайдер

Запросы к языковой модели выполняются через [OpenAI API](https://platform.openai.com/docs/api-reference/chat) (Chat Completions). В каталоге кода используются переменные окружения `OPENAI_API_KEY` и при необходимости `OPENAI_MODEL` (по умолчанию `gpt-4o-mini`). Файл конфигурации для локального и Docker-запуска: `.env` (шаблон — `.env.example`). Подробности — в `docs/specs/serving-config.md` и `docs/specs/tools-apis.md`.

### Edge cases на демо

* Неоднозначные запросы — уточняющие вопросы перед подбором
* Пустая выдача по каталогу — объяснение и общие рекомендации без «сырой» ошибки
* Сбои и некорректные ответы LLM — повтор запроса или fallback на безопасный шаблонный ответ

## Что НЕ делает PoC (out-of-scope)

* Интеграция в продакшн-сайт магазина: единый каталог/SKU, корзина, авторизация, единый UX со страницами товара
* Интеграция с корзиной и системой заказов магазина
* Production-ready масштабирование и обработка высокой нагрузки
* Реальная обработка заказов и доставка товаров
* Интеграция с платежными системами магазина
* Сертификация для production в регулируемой среде

## Что умеет проект

- Показывает чат-виджет на сайте.
- Отвечает на вопросы о товарах.
- Ищет товары в каталоге.
- Сохраняет историю запросов к модели в SQLite.
- Показывает историю запросов из SQLite в Streamlit.
- Хранит каталог товаров в SQLite (`back/database/products.db`, таблица `products`).

## Два флоу

Бот работает в двух режимах:

1. Подбор товаров под задачу (`task`).
2. Ответы на вопросы о товарах (`consultation`).

### 1. Подбор товаров

Что делает:

- понимает задачу пользователя;
- может задать уточняющий вопрос;
- подбирает нужные товары;
- показывает подходящие позиции из каталога.

Пример:

1. Пользователь пишет: `Нужно поклеить обои в комнате`.
2. Бот понимает, что это задача на подбор товаров.
3. Если нужно, бот задаёт уточнение.
4. Потом показывает, что может понадобиться: обои, клей, грунтовка, валик, шпатель.

### 2. Ответы на вопросы

Что делает:

- понимает, что пользователь спрашивает про товар;
- отвечает коротко и по делу;
- при необходимости показывает товары из каталога.

Пример:

1. Пользователь пишет: `Что лучше для стен: валик или кисть?`
2. Бот понимает, что это вопрос-сравнение.
3. Отвечает, чем отличаются варианты и что выбрать в зависимости от задачи.
4. Если товары есть в магазине, показывает их карточки ниже ответа.

## Что где лежит

### `back/`

- Код для анализа запросов, поиска товаров и работы с моделью - здесь лежит логика, которая решает, что ответить пользователю.
- Панель Streamlit - показывает историю запросов и LLM-логи.
- Каталог товаров в SQLite - используется для поиска подходящих товаров.

### `widget/`

- Сервер виджета и API чата - отдаёт страницу чата и обрабатывает сообщения.
- Статические файлы виджета - это `loader.js`, `chat.js` и стили.
- Шаблон чата - HTML-страница, внутри которой живёт чат.
- Настройки доменов - определяют, где виджет можно открывать.

## Как запустить в Docker

### 1. Скопируйте `.env`

```bash
cp .env.example .env
```

Вставьте **API-ключ OpenAI** ([platform.openai.com](https://platform.openai.com/api-keys)). При необходимости задайте модель (`OPENAI_MODEL`, по умолчанию в коде — `gpt-4o-mini`).

```env
OPENAI_API_KEY=your_openai_api_key_here
# OPENAI_MODEL=gpt-4o-mini
# Необязательно: путь к SQLite-базе каталога
# PRODUCTS_DB_PATH=back/database/products.db
# Необязательно: имя таблицы с товарами
# PRODUCTS_TABLE=products
```

### 2. Соберите и запустите контейнер

```bash
docker compose up -d --build
```

### 3. Откройте ссылки

- `http://localhost:8001` - демо-страница с виджетом.
- `http://localhost:8501` - панель с логами и запросами.

## Каталог товаров

По умолчанию проект читает товары из SQLite-базы `back/database/products.db`, таблица `products`.
