# Spec: Observability / Evals

## Что собирается

| Данные | Источник | Где смотреть |
|--------|---------|-------------|
| Количество LLM-вызовов | Логи (глобальный счётчик) | Streamlit Dashboard |
| Latency каждого LLM-вызова | Логи (поле duration) | Streamlit Dashboard |
| Промпты и ответы LLM | Логи (system_prompt, user_prompt, response) | Streamlit Dashboard |
| Имя промпта (`prompt_name`) | Централизованный registry промптов → callback → лог | Streamlit Dashboard |
| Группировка вызовов по запросу пользователя | Логи (структура user_requests) | Streamlit Dashboard |
| Токены на вызов (`prompt_tokens`, `completion_tokens`) | Поле `usage` ответа OpenAI → callback → логирование в SQLite | Streamlit Dashboard (развёртка по вызову) |
| Оценочная стоимость вызова в USD (`cost_usd`) | Расчёт в API по тарифам входа/выхода для выбранной модели | Streamlit Dashboard (на вызов, на пользовательский запрос, сумма по логу) |
| HTTP-события (создание сессий, ошибки) | stdout (uvicorn) | Терминал / Docker logs |

Нет: Prometheus, Grafana, OpenTelemetry, distributed tracing.

## Логи LLM-вызовов

Логи LLM-вызовов хранятся в SQLite и доступны API и Dashboard.

- Хранит последние **20 групп** пользовательских запросов (старые удаляются)
- Каждая группа содержит все LLM-вызовы, сделанные в рамках одного запроса пользователя
- Для каждого LLM-вызова сохраняются `function`, `prompt_name`, `system_prompt`, `user_prompt`, `response`
- **Не логируется:** персональные данные, API-ключи, IP-адреса, идентификаторы сессий

## Токены и стоимость

После каждого успешного вызова Chat Completions в лог попадают числа из `response.usage`: **prompt** и **completion** токены. Для каждой записи LLM-вызова в SQLite сохраняются поля `prompt_tokens`, `completion_tokens` и рассчитанное **`cost_usd`** (оценка по зашитым в код тарифам за токен; при смене модели или цен OpenAI константы в приложении нужно обновлять вручную, иначе сумма расходится с биллингом провайдера).

В **Streamlit** по данным LLM-логов показываются: стоимость и токены **на один LLM-вызов**, агрегаты **на один пользовательский запрос** (сумма по всем вызовам в группе) и **суммарная стоимость / токены** по отображаемым в сессии дашборда запросам. Отдельных алёртов по порогу стоимости и экспорта в биллинг-витрину нет — только просмотр в UI.

## Трейсы

Распределённого трейсинга нет. Логическая цепочка восстанавливается по группировке вызовов в LLM-логах — все вызовы одного пользовательского запроса связаны в одну группу.

## Evals

| Метрика | Цель | Метод |
|---------|:----:|-------|
| Точность классификации типа запроса | > 90% | Ручная разметка ~50 примеров |
| Релевантность рекомендаций | > 75% | Экспертная оценка |
| Доля запросов без ошибок | > 90% | Подсчёт exceptions в логах |
| Latency p95 | < 20 сек | Замер через Dashboard |
| Throughput | > 10 req/min | Нагрузочный тест |

За рамками PoC: A/B тестирование промптов, LLM-as-a-judge, онлайн-оценка.
