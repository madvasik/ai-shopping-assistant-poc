# Spec: Retriever

## Источники данных

SQLite-файл `products.db`, таблица `products` (конфигурируется через env). Доступ read-only. Поля: `title`, `category`, `description`, `price`, `price_currency`, `search_text`.

## Индекс

При первом запросе каталог загружается из SQLite в DataFrame, строится BM25Okapi-индекс по полю `search_text` (= title + category + description). Индекс хранится in-memory, инициализируется один раз.

Токенизация: разбивка по словам (`\b\w+\b`), lower case. Нет стемминга, нет лемматизации.

## Поиск и скоринг

1. BM25 raw scores по токенам запроса
2. Стоп-слова удаляются из запроса перед подсчётом бонусов
3. **Бонус** за полное совпадение всех значимых слов: `+3.0 × N слов`
4. **Штраф** за частичное совпадение: `-3.0` (≥50% слов) или `-10.0` (<50% слов)
5. Простая нормализация окончаний для гибкого сравнения

## Reranking

После BM25 — LLM-фильтр: проверяет соответствие каждого товара названию категории поиска. Возвращает бинарный вектор. При ошибке LLM — все товары считаются релевантными.

## Параметры вызовов

| Контекст | top_k | Порог score | Итог |
|----------|:-----:|:-----------:|------|
| Поиск по сущности / названию товара | 10 | > 0.3 | top 3 на категорию |
| Fallback (нет результатов) | 60 | нет | top 5 |
| Проверка тематики | 5 | > 0.3 | только факт наличия |

## Ограничения

- Только лексический поиск: синонимы и опечатки не обрабатываются
- Нет кэша поисковых запросов
- Индекс не обновляется без перезапуска процесса
