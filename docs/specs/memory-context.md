# Spec: Memory / Context

## Session State

Сессия хранится in-memory в FastAPI-процессе. Содержит: идентификатор виджета, хост страницы, время создания, историю диалога.

Создание: явно через `POST /api/session` или авто-создание при первом сообщении (только для localhost). Клиент хранит идентификатор сессии в localStorage браузера.

**Ограничения:** нет TTL, нет GC — возможна утечка памяти при долгой работе без перезапуска. Нет поддержки нескольких инстансов.

## Memory Policy

| Параметр | Значение | Обоснование |
|---------|---------|-------------|
| Макс. сообщений в истории | 20 (LIFO) | Ограничение RAM; достаточно для контекста |
| Персистентность между сессиями | Нет | Privacy; PoC scope |
| Персистентность при перезапуске | Нет | Нет внешнего хранилища |
| Хранение персональных данных | Нет | 152-ФЗ |

## Context Budget

| Шаг | Что передаётся в LLM |
|-----|---------------------|
| Проверка тематики | Только текущее сообщение |
| Классификация типа | Последние 5 сообщений |
| Извлечение товаров из запроса | Только текущее сообщение |
| Генерация экспертного ответа | Только текущее сообщение |
| Уточняющий вопрос | Полная история сессии |
| Определение списка товаров | Только текущее сообщение (или enhanced_query) |
| Фильтр релевантности | Список товаров (не история диалога) |

## Follow-up

При ответе на уточняющий вопрос оркестратор строит `enhanced_query`:

`{исходная задача}. {уточняющий вопрос} Ответ: {ответ пользователя}`

Этот запрос передаётся в LLM вместо последнего сообщения.
