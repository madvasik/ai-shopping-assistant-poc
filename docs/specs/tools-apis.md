# Spec: Tools / APIs

## OpenAI API

Sync-клиент (`openai.OpenAI`), эндпоинт Chat Completions. Нет встроенного retry в коде приложения. Timeout — по умолчанию библиотеки.

### Контракты LLM-вызовов

| Назначение | Вход | Выход | max_tokens | Fallback |
|-----------|------|-------|:----------:|----------|
| Проверка тематики | текст запроса | `"да"` / `"нет"` | 10 | `False` |
| Классификация типа | последние 5 сообщений | `"task"` / `"consultation"` | 10 | `"task"` |
| Извлечение товаров | текст запроса | JSON-массив строк | 200 | `[]` |
| NLU / слоты (`nlu_with_llm`) | последние сообщения + слоты | JSON слотов | 200 | текущие слоты |
| Фильтр релевантности | название категории + список товаров | бинарная строка `"011"` | 10 | все = 1 |
| Экспертный ответ | текст запроса (+ контекст каталога в `answer`) | текст | 1000 | `None` |
| Уточняющий вопрос | текст запроса + история | текст вопроса или null | 150 | `None` |
| Список товаров для задачи | текст запроса | JSON с текстом и списком | 2000 | `[]` |

**Side effects:** каждый вызов записывается в лог через callback; из `response.usage` в лог попадают токены и оценочная стоимость вызова (USD); тарифицируется OpenAI по своим правилам. Подробнее — `docs/specs/observability-evals.md` (раздел «Токены и стоимость»).

**Защита:** `OPENAI_API_KEY` только через env; `max_tokens` ограничивает вывод (защита от prompt injection); ответы валидируются по ожидаемому формату — при несоответствии используется дефолт.

## SQLite Product Catalog

Read-only доступ. Таблица `products` (конфигурируется через env). Пользовательский ввод в SQL не передаётся — имя таблицы валидируется регулярным выражением.

## Widget REST API

| Endpoint | Запрос | Ответ | Ошибки |
|----------|--------|-------|--------|
| `POST /api/session` | `{widget_key, page_url}` | `{session_id}` | 403, 404, 429 |
| `POST /api/chat` | `{session_id, message, context, widget_key}` | `{reply}` | 400, 403, 429, 502 |
| `GET /health` | — | `{status, time}` | — |

**Rate limit:** 30 запросов за 5 минут на сессию / IP (sliding window). При превышении — HTTP 429 с `retry_after_seconds`.

**Timeout (upstream режим):** 10 сек общий, 3 сек на подключение. В локальном режиме (default) явного timeout нет.
