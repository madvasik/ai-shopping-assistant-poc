# Governance: AI Shopping Assistant

## Risk Register

| Риск | Вероятность/влияние | Детект | Защита | Остаточный риск |
|------|------------------------|--------|--------|----------------|
| Ошибки LLM в классификации интентов | Средняя/Среднее | Логирование ответов LLM, проверка формата ответа | Валидация ответов, fallback на дефолтное значение при ошибке | Низкий |
| Недоступность LLM API | Низкая/Высокое | Мониторинг статуса API, логирование ошибок подключения | Retry логика, fallback на базовые ответы, кеширование запросов | Средний |

| Prompt injection атаки | Низкая/Среднее | Валидация входных данных, мониторинг подозрительных паттернов в запросах | Санитизация промптов, ограничение длины запросов, fallback при подозрительных запросах | Низкий |

| Низкая производительность при росте нагрузки | Средняя/Среднее | Мониторинг времени ответа, логирование производительности | Ограничение нагрузки, оптимизация запросов, масштабирование (для production) | Средний (для PoC) |

## Политика логов

### Что логируется

* **Все запросы к LLM** логируются через модуль логирования:
  - Название функции, которая делает запрос
  - Полный промпт, отправленный в LLM (с заменой персональных данных на плейсхолдеры)
  - Полный ответ LLM (с заменой персональных данных на плейсхолдеры)
  - Время выполнения запроса
  - Статус запроса (успех/ошибка)

### Что НЕ логируется

* **Персональные данные пользователей** (имена, телефоны, адреса, email) - перед логированием заменяются на плейсхолдеры (например, `[ИМЯ]`, `[ТЕЛЕФОН]`, `[АДРЕС]`)
* **Пароли и токены доступа к API** - не включаются в логи

### Хранение логов

* Для PoC: логи хранятся локально в памяти приложения
* Логи отображаются в боковой панели Streamlit приложения
* В production: рекомендуется хранить логи в отдельной системе

---

## Работа с персональными данными

### Сбор данных

* Бот **НЕ собирает** персональные данные явно
* Запросы пользователей могут содержать косвенные персональные данные (например, адреса в контексте задачи), но они не сохраняются
* Система не хранит историю диалогов между сессиями (для PoC)
* Бот не требует регистрации или авторизации пользователей

### Рекомендации для production

* Соответствие 152-ФЗ «О персональных данных»
* Получение согласия пользователя на обработку персональных данных
* Минимизация сбора и хранения персональных данных
* Прозрачная политика конфиденциальности

## Защиты от injection

### Валидация входных данных

* Проверка формата запросов пользователя
* Ограничение длины запросов (максимум символов)
* Фильтрация подозрительных паттернов (например, попытки injection в промпты)

### Санитизация промптов

* Санитизация промптов перед отправкой в LLM API
* Удаление потенциально опасных символов и паттернов
* Ограничение контекста промпта для предотвращения injection

### Fallback механизмы

* При обнаружении подозрительных паттернов в запросе - fallback на базовые ответы
* При ошибке LLM - повторный запрос или fallback на дефолтные ответы
* Логирование всех подозрительных случаев для анализа

## Подтверждение действий

### Что делает бот

Бот является консультантом и **только рекомендует** товары. Бот не выполняет никаких действий, связанных с управлением магазином:

* Бот **НЕ создает** заказы на товары
* Бот **НЕ обрабатывает** платежи
* Бот **НЕ изменяет** данные в системах магазина
* Бот **НЕ отправляет** уведомления пользователям
* Бот **НЕ имеет доступа** к корзине или системе заказов магазина

* Все действия бота - только:
  - Консультации по товарам
  - Рекомендации товаров на основе запроса пользователя
  - Отображение информации о товарах из каталога
  - Генерация ответов и советов через LLM

### Роль бота

Бот работает как независимый консультант, который помогает пользователям найти нужные товары. Финальное решение о покупке и добавлении товаров в корзину принимает сам пользователь на сайте магазина. Бот только предоставляет информацию и рекомендации.
