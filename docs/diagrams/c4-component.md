# Диаграмма компонентов — Оркестратор

```mermaid
flowchart TD
    api["API-слой"]
    llm["LLM"]
    catalog[("Каталог товаров")]

    subgraph orchestrator["Оркестратор"]
        guard["Проверка тематики"]
        router["Определение типа запроса"]

        subgraph consult["Ветка: Консультация"]
            c1["Извлечение товаров из запроса"]
            c2["Генерация экспертного ответа"]
            c3["Поиск по каталогу"]
            c4["Фильтр релевантности"]
        end

        subgraph task["Ветка: Задача"]
            t1["Уточняющий вопрос (если нужен)"]
            t2["Определение списка нужных товаров"]
            t3["Поиск по каталогу"]
            t4["Фильтр релевантности"]
        end

        result["Формирование ответа"]
    end

    api --> guard
    guard -->|"не по теме"| result
    guard -->|"по теме"| router
    router -->|"консультация"| c1
    router -->|"задача"| t1
    c1 --> c2 --> c3 --> c4 --> result
    t1 --> t2 --> t3 --> t4 --> result
    consult -->|"LLM-вызовы"| llm
    task -->|"LLM-вызовы"| llm
    c3 --> catalog
    t3 --> catalog
    result --> api
```

## Параметры

| Параметр | Значение |
|-----------|:-------:|
| Товаров из поиска до фильтрации | 60 |
| Товаров в финальном ответе | 5 |
| Максимум уточняющих вопросов | 2 |
